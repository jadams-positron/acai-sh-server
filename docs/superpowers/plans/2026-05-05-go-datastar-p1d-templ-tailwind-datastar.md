# Go + Datastar Rewrite — Phase 1d: templ + Tailwind v4 + Datastar Runtime

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Replace the bare-HTML auth pages with templ-rendered pages styled by Tailwind v4 (standalone CLI) and made reactive via vendored Datastar runtime. Same routes, same flow — better presentation. After P1d, the login/register/teams pages look like a modern web app and the toolchain (templ generate → tailwind → go build) is wired into both `just dev` and CI.

**Architecture:** templ files (`*.templ`) compile to Go (`*_templ.go`) at build time. Tailwind v4 standalone CLI scans templ + Go sources for class usage and emits `assets/dist/app.css`. `go:embed` packs the CSS, the Datastar runtime, and a small heroicon SVG subset into the binary. Handlers swap from `html/template.Execute` to `componentName(...).Render(ctx, w)`.

**Tech Stack:** `github.com/a-h/templ` (and its CLI `templ generate`), Tailwind v4 standalone CLI (`tailwindcss-macos-arm64` / `tailwindcss-linux-x64`), vendored Datastar runtime (downloaded once into `assets/js/`), heroicons SVG subset.

**Reference:** Spec §6 (Frontend & Datastar). Branch `rewrite/go-datastar` after P1c at commit `0db9252`.

---

## Scope decisions

- **Three pages converted in P1d:** login, register, and the /teams stub. The login-requested + register-requested confirmation pages get the same shared layout. That's it for P1d.
- **Datastar interactions kept minimal in P1d.** The login form gets `data-on-submit__prevent` + `data-on-submit="@post('/users/log-in')"` for SPA-feel submit, with the response patched into a result fragment. Validation and reactive UX expand in P3 when real pages land.
- **No tailwind config file.** Tailwind v4 is config-free; we use `@source` directives in `assets/css/app.css` to point at templ + Go source dirs.
- **Toolchain installed via `go install` (templ) + downloaded binary (tailwindcss).** The `tailwindcss` standalone CLI is downloaded as part of `just install-tools` and committed to `bin/tailwindcss-{os}-{arch}` is **avoided** — instead we add a `just install-tools` recipe that downloads and installs the binary into `$(go env GOPATH)/bin`. CI runs the same recipe.

```go
package handlers

import (
	"io/fs"
	"net/http"

	"github.com/go-chi/chi/v5"

	"github.com/jadams-positron/acai-sh-server/assets"
)

// MountStatic serves vendored + generated assets at /_assets/* from the
// embedded FS. Caching headers are conservative for now (5min); P3 can tune.
func MountStatic(r chi.Router) {
	sub, err := fs.Sub(assets.FS, ".")
	if err != nil {
		// embed.FS guarantees this never fails for an existing root.
		panic("assets: fs.Sub: " + err.Error())
	}
	fileServer := http.FileServer(http.FS(sub))
	r.Get("/_assets/*", http.StripPrefix("/_assets/", cacheHeaders(fileServer)).ServeHTTP)
}

// cacheHeaders adds a 5-minute Cache-Control to static asset responses.
func cacheHeaders(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		w.Header().Set("Cache-Control", "public, max-age=300")
		next.ServeHTTP(w, r)
	})
}
```

## Task 3: Toolchain — `just install-tools`, `just gen`, CI integration

- [ ] **Step 1:** Update `justfile` — add `install-tools` and templ/css recipes; update `gen`:

```just
# install-tools installs templ, sqlc, and tailwindcss into $GOPATH/bin (or PATH).
# Run once per fresh clone. CI runs this before `just gen` and `just lint`.
install-tools:
    go install github.com/a-h/templ/cmd/templ@v0.3.1001
    go install github.com/sqlc-dev/sqlc/cmd/sqlc@v1.31.1
    @echo "Installing tailwindcss standalone CLI..."
    @if ! command -v tailwindcss >/dev/null 2>&1; then \
      OS=$$(uname -s | tr '[:upper:]' '[:lower:]'); \
      ARCH=$$(uname -m); \
      case "$$ARCH" in \
        x86_64) ARCH=x64 ;; \
        arm64|aarch64) ARCH=arm64 ;; \
      esac; \
      URL="https://github.com/tailwindlabs/tailwindcss/releases/latest/download/tailwindcss-$${OS}-$${ARCH}"; \
      DEST="$$(go env GOPATH)/bin/tailwindcss"; \
      echo "Downloading $$URL -> $$DEST"; \
      curl -fsSL "$$URL" -o "$$DEST"; \
      chmod +x "$$DEST"; \
    fi
    tailwindcss --help >/dev/null 2>&1 || { echo "tailwindcss install failed"; exit 1; }

# gen-templ runs templ generate over all .templ files.
gen-templ:
    templ generate

# gen-css runs tailwind to produce assets/dist/app.css.
gen-css:
    tailwindcss -i assets/css/app.css -o assets/dist/app.css --minify

# gen runs all code/asset generators. Run whenever queries, templates, or styles change.
gen: gen-templ
    sqlc generate
    just gen-css
    @echo "gen: done."
```

(The `@if ! command -v tailwindcss …` guard skips the download if tailwindcss is already in PATH.)

## Risk register

- **Tailwind v4 standalone CLI download URL stability** — The `releases/latest` URL works as long as Tailwind doesn't break the asset naming convention. If CI fails downloading, pin to a specific version (`v4.0.0` or whatever is current).
- **templ + golangci-lint v2** — generated `_templ.go` files have unusual patterns. The new `internal/site/views/.*_templ\.go` exclusion in `.golangci.yml` should suppress noise. If new warnings surface, add to the exclusion list.
- **embed.FS empty `dist/`** — On a fresh clone, `dist/` is empty until `just gen-css` runs. The `dist/.gitkeep` ensures the directory exists; embed accepts empty matches as long as another pattern matches. If `go build` complains, replace `//go:embed dist/* js/* icons/*` with `//go:embed all:dist all:js all:icons`.
- **CI tailwindcss runner OS/arch detection** — runs on `ubuntu-latest` (linux-x64); the `uname -m` returns `x86_64` which we map to `x64`. Verified in plan; double-check on first CI run.
